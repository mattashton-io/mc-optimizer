We need to implement the core mapping logic in `mc-data-prep/agent.py`. 
The agent must ingest raw VMware and other infrastructure CSV sheets (such as within `data/exports/) 

and output fully compliant Google Cloud Migration Center CSV formats: `templates/vminfo.csv` and `templates/diskinfo.csv`.

Review the target schema specifications:
- `templates/vminfo.csv` required columns: MachineId, MachineName, PrimaryIPAddress(optional), PrimaryMACAddress(optional), PublicIPAddress(optional), IpAddressListSemiColonDelimited(optional), TotalDiskAllocatedGiB, TotalDiskUsedGiB, MachineTypeLabel(optional), AllocatedProcessorCoreCount, MemoryGiB, HostingLocation(optional), OsType(optional), OsPublisher(optional), OsName, OsVersion(optional), MachineStatus(optional), ProvisioningState(optional), CreateDate(optional), IsPhysical.
- `templates/diskinfo.csv` required columns: MachineId, DiskLabel, SizeInGib, UsedInGib, StorageTypeLabel.

Generate a plan detailing:
1. How you will parse the source files safely using Pandas.
2. The regex/cleaning strategy to prevent "TypeError: unsupported operand type(s) for /: 'str' and 'int'" when processing numeric string values containing commas (e.g., '1,024.00').
3. How you will map/normalize OS names to 'Linux' or 'Windows' to pass MC imports.
4. How you will generate synthetic MachineIds ('iona-uuid-xxxx') to safely link VMs and disks.
